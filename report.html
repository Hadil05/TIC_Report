2Les réseaux informatiques

1) Réseau (informatique) Network :

Un réseau informatique est un ensemble d’ordinateurs et de périphériques connectés entre eux pour partager des ressources et des informations.
Évolution de Réseau :
Les réseaux informatiques ont été créés dans un premier temps pour permettre la communication entre des terminaux distants et un site central Cependant, l'évolution des technologies a offret des capacités de communication avancées Interconnexion: Un réseau repose sur la notion fondamentale d’interconnexion, qui assure la transmission des données d’un nœud à un autre Communication: Une autre notion fondamentale d’un réseau est la communication, qui permet l’échange de données entre les processus (programmes en exécution).

2) Les caractéristiques de base d’un réseau sont:

la topologie (physique et logique), le débit (en bits/s), la distance maximale (ou portée) et le nombre de nœuds maximum que l’on peut interconnecter.

3) Quelques terminologies en réseau :


Station de travail : Toute machine capable d’envoyer des données vers les réseaux.
Nœud : toute entité pouvant être adressée par un numéro unique.
Paquet: La plus petite unité d’information pouvant être envoyée sur le réseau.
Réseaux hétérogènes: constitué de différents types d'équipements et de technologies.
Débit: Mesure une quantité de données numériques (bits) transmises par seconde.

4) Éléments d'un reseau :


ordinateurs en réseau.
logiciels de communication.
supports de transmission de données.
Les équipements d'interconnexion

5) Classification des réseaux :


1. Par portée l’envergure: LAN :
- LAN est un réseau local d'entreprise avec une portée limitée (quelques centaines de mètres).
-Il est considéré comme un réseau privé et connecte généralement moins de 100 machines
- Les vitesses de transmission vont de 10 à 100 Mb/s. MAN :
-MAN est un réseau métropolitain ou urbain qui relie plusieurs LAN dans une même entreprise ou ville, sur une distance de 10 à 25 km. Le câble co-axial est le support physique le plus utilisé dans ce type de réseau. WAN :
-Les WAN sont des réseaux étendus et publics qui assurent la transmission de données sur de longues distances à l'échelle d'un pays ou de la planète. Il existe d'autres types de réseaux tels que les PAN, WPAN et WLAN, SAN.
2. L’ouverture :
-Les réseaux peuvent être classés par portée ou par les services qu'ils offrent, notamment en réseaux intranet, extranet et internet (publics).

3Le modèle OSI

1)les composants matériel d’un Réseau informatique

-Les cartes réseau : également connues sous le nom de cartes d'interface réseau (NIC), sont des composants matériels essentiels permettant la connexion d'un ordinateur à un réseau, qu'il s'agisse d'Internet ou d'un réseau local. Elles permettent à l'ordinateur d'émettre et de recevoir des données en utilisant des protocoles de communication spécifiques, tels que TCP/IP, qui sont employés pour le transfert de données sur Internet. Les cartes réseau sont installées à l'intérieur de l'ordinateur et sont généralement connectées à une prise Ethernet ou Wi-Fi. Elles jouent un rôle crucial dans la communication entre les ordinateurs d'un réseau.
-switch : ou commutateur réseau, est un composant matériel utilisé pour connecter des appareils au sein d'un réseau local (LAN). Il permet à plusieurs appareils tels que des ordinateurs, des imprimantes, des serveurs, etc., de communiquer entre eux en utilisant des adresses MAC (Media Access Control). Lorsqu'un appareil se connecte au switch, celui-ci vérifie son adresse MAC et l'enregistre dans une table de correspondance. Lorsqu'un autre appareil du réseau souhaite communiquer avec cet appareil, le switch utilise cette table pour envoyer les données uniquement à l'appareil destinataire, évitant ainsi la diffusion à tous les appareils du réseau. Les switches sont disponibles en différentes tailles, adaptées aux réseaux domestiques, aux petits bureaux et aux réseaux d'entreprise plus importants. Ils peuvent offrir des fonctionnalités avancées telles que la gestion des VLAN, la qualité de service (QoS), la gestion du trafic, etc.
-HUB : ou concentrateur, est un composant matériel utilisé pour connecter plusieurs appareils sur un réseau local (LAN). Contrairement à un switch, le hub transmet les données à tous les appareils connectés au lieu de les envoyer uniquement à l'appareil destinataire. Les hubs ont été remplacés par des switches plus efficaces pour la gestion du trafic réseau et la communication rapide et sécurisée entre les appareils. Cependant, les hubs sont encore utilisés dans des réseaux de petite taille ou des applications spécifiques ne nécessitant pas la complexité d'un switch.

2) les modèle OSI

Le modèle OSI (Open Systems Interconnection) est un modèle de référence pour les communications réseau, divisé en sept couches distinctes. Chaque couche a des fonctions spécifiques liées à la communication entre les appareils réseau. Les sept couches du modèle OSI sont les suivantes :
1.Couche physique : transmission des données sur le support physique.
2.Couche liaison de données : transmission des données entre les appareils partageant le même support.
3.Couche réseau : routage des données à travers le réseau.
4.Couche transport : acheminement des données de bout en bout entre les appareils.
5.Couche session : établissement et gestion des sessions entre les appareils.
6.Couche présentation : conversion des données dans un format compréhensible par l'appareil destinataire.
7.Couche application : communication entre les applications sur les appareils réseau.
Bien que le modèle OSI soit théorique et ne soit pas utilisé dans les protocoles de communication réseau modernes tels que TCP/IP, il reste utile pour comprendre les principes fondamentaux de la communication réseau.

3) LES MODÈLES TCP /IP

Le modèle TCP/IP est un modèle de référence pour les communications de réseau qui divise les communications en quatre couches principales : la couche liaison de données, la couche réseau, la couche transport et la couche application. Il est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes, tels que les réseaux d'entreprise, les réseaux Internet et les réseaux sans fil. Il est utilisé avec des protocoles de communication de réseau tels que le protocole Internet (IP), le protocole de contrôle de transmission (TCP) et le protocole de datagramme utilisateur (UDP).

4) L’ adressage IPV 4

L'adressage est crucial dans les communications réseau pour identifier de manière unique les unités en communication. L'adressage IP (Internet Protocol) est utilisé à cet effet. L'adresse IP est un numéro qui identifie chaque ordinateur dans un réseau. Il existe deux modes d'adressage : IPV4 et IPV6.
Dans l'adressage IPV4, chaque paquet contient une adresse source et une adresse destinataire, toutes deux composées de 32 bits (4 octets).
L'adressage IPV6 utilise une codification sur 128 bits. L'adresse est découpée en huit mots de 16 bits séparés par ":".
L'adressage IP permet d'attribuer des identifiants uniques aux ordinateurs dans un réseau, facilitant ainsi l'échange de données à l'aide de protocoles de communication spécifiques.
L'adresse IPV4 est composée de deux parties distinctes : l'adresse réseau et l'adresse hôte. La partie réseau est déterminée par les bits les plus significatifs, tandis que la partie hôte correspond aux bits restants. Il existe différents types d'adresses IPV4 :
1.L'adresse réseau : tous les bits de la partie hôte sont fixés à 0. Cela signifie que tous les appareils d'un même réseau partagent la même adresse réseau.
2.L'adresse de diffusion : tous les bits de la partie hôte sont définis à 1. L'adresse de diffusion permet à un appareil d'envoyer des données à tous les autres appareils du réseau sans les adresser individuellement.
3.Les adresses hôtes : ce sont les adresses attribuées aux appareils individuels du réseau. Elles se situent entre l'adresse réseau et l'adresse de diffusion.
4.Le masque de réseau et de sous-réseaux : le masque de réseau permet de déterminer quels bits de l'adresse IP appartiennent à la partie réseau et à la partie hôte. Il définit la frontière entre ces deux parties de l'adresse.
En utilisant l'adressage IPV4, les appareils d'un réseau peuvent être identifiés de manière unique en utilisant des adresses réseau, des adresses de diffusion, des adresses hôtes et des masques de réseau.

5Systéme de Gestion de Version

1) Qu’est-ce que Git ?

Parmi les logiciels de versioning, ou logiciel de gestion de version , Git est le leader incontesté créé en 2005 par Linus Torvalds,,c’est un logiciel qui permet de conserver un historique des modifications effectuées sur un projet afin de pouvoir rapidement identifier les changements effectués et de revenir à une ancienne version en cas de problème. Les logiciels de gestion de versions sont quasiment incontournables aujourd’hui car ils facilitent grandement la gestion de projets et car ils permettent de travailler en équipe de manière beaucoup plus efficace.

2)A quoi sert un systeme de gestion de version ?

Les systèmes de ges+on de versions perme2ent de gérer les modifica+ons apportées aux projets en conservant un historique des modifica+ons apportées aux fichiers. Cela permet de coordonner le travail de plusieurs personnes sur le même projet, d'éviter les conflits et de s'assurer que tout le monde travaille sur la dernière version du projet. La collabora+on peut être difficile sans système de contrôle de version, et la ges+on des sauvegardes peut rapidement devenir lourde, surtout lorsque plusieurs personnes travaillent sur le même fichier. Git est un logiciel de contrôle de version décentralisé qui vous permet de synchroniser les modifica+ons sur plusieurs machines, de gérer efficacement les conflits et de collaborer sur le même projet en toute sécurité

3) Les deux modèles des logiciels de gestion de version : Modèle centralisé vs modèle décentralisé

Les logiciels de gestion de version construits sur l’un des deux Modèles suivants : le modèle centralisé ou le modèle décentralisé (également appelé distribué). Le modèle centralisé repose sur la centralisation du code source lié au projet sur un serveur central distant, tandis que dans le modèle distribué, le code source du projet est toujours hébergé sur un serveur distant, mais chaque utilisateur est invité à télécharger et à héberger l’intégralité du code source du projet sur sa propre machine. Le modèle distribué, popularisé par Git, présente plusieurs avantages notables par rapport au modèle centralisé.
-Simplicité / flexibilité du travail : il n’est plus nécessaire d’être constamment connecté à un serveur central et l’on peut travailler en ligne sur sa propre machine
- Sécurité : on peut utiliser la copie du projet d’un utilisateur comme sauvegarde en cas de corruption du serveur central.

4) Qu’est-ce que GitHub ?

Git est un logiciel de gestion de version tandis que GitHub est un service en ligne d’hébergement de dépôts Git qui fait office de serveur central pour ces dépôts.

7)Paramétrage de Git

Après avoir installé Git, nous devons le configurer pour éviter de fournir certaines informatons à chaque fois que nous l'utlisons. Cela inclut la saisie du nom d'utlisateur et de l'adresse e-mail que Git utlisera pour chaque projet. Pour ce faire, nous utliserons la commande "git config", qui nous permet de visualiser et de modifier les variables de configuraton qui contrôlent l'apparence et le comportement de Git. Nous ajouterons l'opton "--global" afin que les informatons saisies soient utlisées globalement pour tous les projets Git.
Nous allons donc saisir les commandes suivantes : "git config --global user.name "nom_complet"" et "git config --global user.email your@email.com", en remplaçant " nom_complet " par votre propre nom et "votre @email.com" via votre propre adresse e-mail.
Pour vérifier si les informatons ont été enregistrées, vous pouvez entrer les commandes "git config user.name" et "git config user.email". Les informatons que vous avez saisies doivent apparaître à l'écran.

9)La gestion des informations selon Git

La façon dont Git gère les données est très différente des autres systèmes de contrôle de version. En effet, Git voit les données comme un flux d'instantanés, ou "instantanés". Chaque fois que vous enregistrez l'état d'un projet dans Git, il prend un instantané de l'état du contenu de l'espace de travail et enregistre une référence à cet instantané pour un accès ultérieur.
Chaque instantané est stocké localement dans une base de données sur votre propre machine. L'avantage d'avoir l'historique complet de votre projet localement est que la plupart des opéra+ons Git peuvent être effectuées localement sans se connecter à un serveur central distant. En général, cela rend les opéra+ons plus rapides et plus agréables
